# แนวทางเลือกข้อมูลจาก jhcisdb

เป้าหมายของระบบนี้คือทำ dashboard ภาพรวมจังหวัด ไม่ใช่ย้ายฐานข้อมูล JHCIS ทั้งก้อน ดังนั้นระยะแรกควรดึงเฉพาะข้อมูลสรุป ไม่ดึงชื่อ, เลขบัตรประชาชน, ที่อยู่ละเอียด หรือเบอร์โทรศัพท์

## ชุดข้อมูลขั้นต่ำที่ควรเริ่ม

### 1. สถานะการส่งข้อมูล

ใช้ตรวจว่า 90 แห่งส่งข้อมูลครบหรือไม่

- รหัสหน่วยบริการ
- ชื่อหน่วยบริการ
- วันที่ข้อมูล
- เวลาที่สร้างข้อมูลจาก รพ.สต.
- เวลาที่ส่วนกลางรับข้อมูล

### 2. งานบริการรายวัน

ใช้ดูปริมาณงานของแต่ละ รพ.สต.

- จำนวน visit ทั้งหมด
- จำนวนผู้รับบริการไม่ซ้ำ
- จำนวนผู้ป่วยใหม่ถ้ามี field รองรับ
- จำนวน visit ตามประเภทงาน เช่น OPD, ANC, vaccine, home visit, chronic clinic

### 3. งานโรคเรื้อรัง

ใช้ดูภาระงาน NCD โดยไม่ต้องดึงรายชื่อคนไข้

- จำนวนผู้ป่วย DM ที่มารับบริการ
- จำนวนผู้ป่วย HT ที่มารับบริการ
- จำนวนผู้ป่วย DM หรือ HT ที่มารับบริการแบบไม่ซ้ำ
- จำนวน visit คลินิกโรคเรื้อรัง
- จำนวนที่ได้รับการวัดความดัน
- จำนวนที่มีผลตรวจน้ำตาล เช่น FBS
- จำนวน visit ที่ไม่มี diagnosis เพื่อดูคุณภาพข้อมูล NCD

ตัวอย่างนิยามระยะแรก:

- DM: diagnosis ICD-10 กลุ่ม `E10-E14`
- HT: diagnosis ICD-10 กลุ่ม `I10-I15`
- DM/HT: ผู้รับบริการไม่ซ้ำที่มี diagnosis DM หรือ HT ในวันรายงาน
- BP screened: visit ที่มีค่าความดัน systolic หรือ diastolic
- FBS screened: visit หรือ lab order ที่มีรายการ FBS/fasting blood sugar

### 4. งานแม่และเด็ก

- จำนวน ANC visit
- จำนวนหญิงตั้งครรภ์ที่มารับบริการ
- จำนวน vaccine visit
- จำนวนเด็ก 0-5 ปีที่รับบริการ

### 5. งานเยี่ยมบ้านและชุมชน

- จำนวน home visit
- จำนวนผู้ป่วยติดบ้าน/ติดเตียงที่ได้รับเยี่ยม
- จำนวนกิจกรรมชุมชน ถ้าฐานมี table รองรับ

### 6. ส่งต่อและฉุกเฉิน

- จำนวน refer out
- จำนวน emergency case
- จำนวน case ที่ส่งต่อแยกตามปลายทาง ถ้าต้องการ

### 7. คุณภาพข้อมูล

ใช้ช่วย สสจ. เห็นปัญหาการบันทึกข้อมูล

- จำนวน visit ที่ไม่มี diagnosis
- จำนวน visit ที่ไม่มีรหัสสิทธิ์หรือแผนก
- วันที่ visit ล่าสุดในฐาน
- จำนวน record ที่แก้ไข/เพิ่มในวันล่าสุด ถ้ามี timestamp

## ตัวชี้วัดระยะแรกที่แนะนำ

ให้เริ่มด้วย 12 ตัวนี้ก่อน เพราะทำ dashboard ได้เร็วและเสี่ยงน้อย:

1. จำนวน รพ.สต. ที่ส่งข้อมูลแล้ว
2. จำนวน visit ทั้งหมด
3. จำนวนผู้รับบริการไม่ซ้ำ
4. จำนวน chronic follow-up
5. จำนวน DM patients
6. จำนวน HT patients
7. จำนวน DM/HT patients แบบไม่ซ้ำ
8. จำนวน BP screened
9. จำนวน FBS screened
10. จำนวน ANC visit
11. จำนวน vaccine visit
12. จำนวน home visit
13. จำนวน refer out
14. จำนวน emergency case
15. จำนวน visit ที่ไม่มี diagnosis
16. วันที่ข้อมูลล่าสุดของแต่ละ รพ.สต.
17. เวลาที่ส่วนกลางรับข้อมูลล่าสุด

## ตาราง JHCIS ที่ควรสำรวจก่อน

ชื่อ table อาจต่างตาม version จึงให้สำรวจจาก `jhcisdb` จริงก่อน โดยค้นหากลุ่ม table/column เหล่านี้:

- `person`, `patient`, `pid`, `cid`, `hn`
- `visit`, `service`, `vstdate`, `date_serv`
- `diag`, `diagnosis`, `icd10`
- `chronic`, `clinic`, `ncd`, `dm`, `ht`
- `anc`, `preg`, `prenatal`
- `epi`, `vaccine`, `immun`
- `home`, `community`
- `refer`, `er`, `emergency`
- `drug`, `lab`

## วิธี map เข้าระบบนี้

ระบบกลางรับข้อมูลรูปแบบนี้:

```json
{
  "facility_id": "10001",
  "facility_name": "RPST Sample",
  "report_date": "2026-07-05",
  "total_visits": 120,
  "unique_patients": 98,
  "chronic_followups": 24,
  "ncd_dm_patients": 18,
  "ncd_ht_patients": 22,
  "ncd_dm_ht_patients": 31,
  "ncd_bp_screened": 80,
  "ncd_fbs_screened": 25,
  "missing_diagnosis": 4,
  "anc_visits": 6,
  "vaccine_visits": 18,
  "home_visits": 9,
  "refer_out": 3,
  "emergency_cases": 1,
  "source_generated_at": "2026-07-06T00:15:00Z",
  "payload": {
    "ncd_definition": "DM=E10-E14, HT=I10-I15",
    "top_diagnosis": [
      { "icd10": "I10", "count": 12 }
    ]
  }
}
```

ค่าหลัก 8 ค่าแรกเก็บเป็น column ในฐานกลาง ส่วนตัวชี้วัดเสริมใส่ใน `payload` ได้ก่อน เมื่อใช้งานจริงแล้วค่อยแยก column เพิ่ม

## ข้อมูลที่ไม่ควรส่งเข้ากลางในระยะแรก

- ชื่อ-นามสกุล
- เลขบัตรประชาชน
- ที่อยู่ละเอียด
- เบอร์โทรศัพท์
- เลข HN รายบุคคล
- รายการยา/lab รายคนแบบระบุตัวบุคคล

ถ้าจำเป็นต้องใช้ข้อมูลระดับบุคคลภายหลัง ควรออกแบบเรื่อง PDPA, สิทธิ์เข้าถึง, audit log และการเข้ารหัสก่อน
